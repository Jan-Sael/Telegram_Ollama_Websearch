# Telegram + Ollama Web Search Bot

A single-file Python application that lets one authorized Telegram user chat with a local Ollama model.

The bot uses:

- Telegram Bot API for messaging
- Ollama `/api/chat` for local LLM inference
- `reporter-14b:latest` as the configured model
- A free DuckDuckGo HTML-based web search tool
- Python standard library only

No `.env` file, `requirements.txt`, database, or third-party Python package is required.

---

## Project Structure

```text
WeeklyPlantUpdate/
├── main.py
└── README.md
```

The application logic and configuration are all inside `main.py`.

## AI Tools

The model has one callable tool:

```text
web_search(query, max_results)
```

The model can decide when to use web search, for example for:

- Current information
- Recent news
- Today's events
- Current prices
- Latest software versions
- Public developments
- Explicit requests to search or verify something online

The current implementation returns search result titles, URLs, and snippets.

Important limitation: the bot does not currently have a webpage-reading/fetch tool. It searches the web but does not open and extract full article pages.

### 2. Ollama

Ollama must be running and reachable by `main.py`.

Default configuration:

```python
OLLAMA_URL = "http://localhost:11434"
```

This is correct only when Ollama is running on the same Windows machine as `main.py`.

If Ollama runs on another machine, such as an Ubuntu server, configure its reachable address instead:

```python
OLLAMA_URL = "http://192.168.1.100:11434"
```

Replace `192.168.1.100` with the actual server IP.

Do not expose Ollama directly to the public internet.

### 3. Ollama Model

The configured model is:

```text
reporter-14b:latest
```

Verify that it exists:

```powershell
ollama list
```

If the model already exists, you should see `reporter-14b:latest`.

Test it directly:

```powershell
ollama run reporter-14b:latest
```

If the model is on a remote Ubuntu server, run those commands on that server.

### 4. Telegram Bot

Create a bot through Telegram's official `@BotFather`.

Basic process:

1. Open Telegram.
2. Search for `@BotFather`.
3. Start a chat.
4. Send `/newbot`.
5. Follow the prompts.
6. Copy the generated bot token.
7. Keep the token secret.

If a token has been exposed in chat, source code, screenshots, logs, or a public repository, revoke/regenerate it before deployment.

## Configuration

Open `main.py` and edit the configuration section near the top.

### Telegram Bot Token

Set:

```python
TELEGRAM_BOT_TOKEN = "PUT_YOUR_NEW_TELEGRAM_BOT_TOKEN_HERE"
```

Example format only:

```python
TELEGRAM_BOT_TOKEN = "1234567890:EXAMPLE_TOKEN"
```

Do not use the example value.

### Allowed Telegram User

The bot is restricted to:

```python
ALLOWED_TELEGRAM_USER_ID = 7195406693
```

Only that Telegram user ID is allowed to interact with the bot.

### Ollama Endpoint

For Ollama on the same PC:

```python
OLLAMA_URL = "http://localhost:11434"
```

For Ollama on another LAN machine:

```python
OLLAMA_URL = "http://SERVER_IP:11434"
```

### Model

Set:

```python
OLLAMA_MODEL = "reporter-14b:latest"
```

## How the Agent Loop Works

```text
Telegram User
    |
    v
Telegram Bot API
    |
    v
main.py
    |
    v
Ollama /api/chat
    |
    +---- final answer ----------------------+
    |                                        |
    +---- tool call: web_search              |
             |                               |
             v                               |
       DuckDuckGo HTML search                |
             |                               |
             v                               |
       Search result JSON                    |
             |                               |
             v                               |
       Ollama receives tool result           |
             |                               |
             +-------------------------------+
                             |
                             v
                    Telegram response
```

The model decides whether to call the tool. Python executes the requested tool and sends the result back to the model.

## Conversation Memory

Conversation history is stored only in RAM:

```python
conversation_history = defaultdict(list)
```

Default limit:

```python
MAX_HISTORY_MESSAGES = 20
```

Consequences:

- Memory is lost when `main.py` stops.
- `/clear` removes the current stored chat history.
- There is no database.
- There is no persistent conversation archive implemented by this application.

Telegram itself still processes/stores bot-chat messages according to Telegram's service architecture and policies.

## Security Notes

### Telegram Bot Chats Are Not End-to-End Encrypted

Telegram bot conversations are not E2EE Secret Chats.

Do not assume bot messages are suitable for:

- Passwords
- API keys
- Highly confidential company information
- Sensitive personal documents
- Secrets
- Private credentials

### Protect the Bot Token

Anyone with the bot token may be able to control the bot through the Telegram Bot API.

If exposed:

1. Open `@BotFather`.
2. Revoke/regenerate the token.
3. Replace the token in `main.py`.
4. Restart the application.

### User Restriction

The application checks:

```python
ALLOWED_TELEGRAM_USER_ID = 7195406693
```

Messages from other user IDs are rejected.

This access control is useful, but it does not make Telegram bot communication end-to-end encrypted.

### Ollama Network Exposure

Avoid exposing:

```text
11434/tcp
```

directly to the public internet.

Prefer:

- Same-machine access
- Private LAN
- WireGuard VPN
- Firewall allow-listing
- A properly authenticated reverse proxy if remote public access is unavoidable

## Troubleshooting

### Error: Ollama Unreachable

Example:

```text
Status: Ollama unreachable
```

Check:

```powershell
Invoke-RestMethod http://localhost:11434/api/tags
```

If Ollama is remote, replace `localhost` with the server IP.

Possible causes:

- Ollama is not running.
- Wrong IP address.
- Wrong port.
- Firewall blocking port 11434.
- Ollama listening only on localhost.
- Docker port not published.
- Windows cannot route to the Ubuntu server.

### Model Not Found

Check:

```powershell
ollama list
```

The configured name must match exactly:

```text
reporter-14b:latest
```

### Telegram Token Rejected

Regenerate the token through `@BotFather`, update:

```python
TELEGRAM_BOT_TOKEN = "..."
```

and restart `main.py`.

### Bot Does Not Respond

Check the PowerShell console for:

```text
[ERROR]
[NETWORK ERROR]
[POLLING ERROR]
[HTTP ERROR]
```

Also verify:

- `main.py` is still running.
- Internet access is available.
- Telegram API is reachable.
- The sender's Telegram user ID matches the configured allowed ID.

### Web Search Returns No Results

The implementation uses DuckDuckGo's HTML interface.

Possible causes:

- HTML structure changed.
- Temporary rate limiting.
- Network filtering.
- Search endpoint behavior changed.
- Regex parser no longer matches the returned HTML.

This search method is free but less stable than a formal search API.

### Tool Is Never Called

The model itself must support and correctly follow Ollama tool-calling behavior.

Possible causes:

- The custom model does not reliably support tool calling.
- The model decides the query does not require web search.
- The model's template does not preserve tool-calling behavior.
- The custom `reporter-14b:latest` model configuration interferes with tool calls.

Test with an explicit request:

```text
Use web_search to find the latest Ollama news today.
```

Watch the console for:

```text
[TOOL CALL]
```

## Current Limitations

- Text messages only
- One authorized Telegram user
- One AI tool: `web_search`
- No full webpage fetching
- No image understanding
- No voice messages
- No document parsing
- No persistent memory
- No database
- No streaming Telegram response
- DuckDuckGo HTML search may change or rate-limit
- Tool calling depends on model capability
- Bot chats are not end-to-end encrypted
